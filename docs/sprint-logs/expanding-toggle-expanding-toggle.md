# Sprint Log: expanding-toggle

**Plan:** docs/sprint-plans/expanding-toggle.md
**Sprint goal:** Replace the per-table 36×20 pill switch with a 10×10 dot anchored at the table's top-right corner (8px overlap, 2px overhang) that expands into a 28×16 compact switch on mouse hover or touch tap, with a 7px transparent hit-target buffer on every side and the softer `#3d85c6` accent color.
**Date:** 2026-06-01
**Result:** Completed

## Attempts

### Attempt 1
- **Developer (Sonnet):** Introduced the 12 named geometry constants at module scope. Replaced the `<label class="dr-ext-toggle">+<input type=checkbox>+<span class="dr-ext-toggle-slider">` DOM with a single `<button class="dr-ext-morph" type="button" aria-pressed="false" aria-label="Toggle rounding for table">` wrapping `<span class="dr-ext-morph-visible"><span class="dr-ext-morph-knob"></span></span>`. Rewrote the CSS injected by `ensureToggleStyleInjected` so every geometry literal is interpolated from a constant, including a `@media (hover: hover) and (pointer: fine)` rule for mouse hover-grow and a `:focus-visible` rule that mirrors the expanded state for keyboard. Updated `syncSwitchForTable` to set `aria-pressed` on the stored button. Added the touch-aware `pointerdown` (capture pointer type) → `click` (mouse/keyboard: toggle immediately; touch/pen: first tap expands without toggling, second tap toggles, with a `TOUCH_AUTOCOLLAPSE_MS = 3000` auto-collapse timer) handler. Added a module-level `_globalTapCollapseAdded` guard plus a single `document.addEventListener('pointerdown', …)` that removes `.expanded` from any morph button the tap landed outside of.
- **Test-writer (Sonnet, adversarial):** Updated `injectToggleEntry` and related fixtures to construct proper button stubs (with `setAttribute`/`getAttribute`/`classList`/`contains`/`addEventListener`/event-dispatch). Removed the tests that asserted the old DOM (`<input type=checkbox>` / `.dr-ext-toggle` class / explicit Enter-keydown listener) and replaced them with new tests covering: DOM shape and accessible name; anchor geometry both at scroll=0 and scroll≠0 in terms of the exposed constants; `syncSwitchForTable` flipping `aria-pressed` for fresh / rounded / showing-original states; mouse click immediate toggle; touch / pen first-tap-expand and second-tap-toggle; auto-collapse timer (faked `setTimeout`); tap-outside collapse and tap-inside preservation; knob-travel derivation; and a static check that every geometry constant name is referenced inside `ensureToggleStyleInjected` and `positionToggle`.
- **Reviewer verdict (Opus):** **BLOCK** — the plan's §3.6 `wrapperLeft` formula double-counts padding (`TOGGLE_DOT_PX + 2 * padding`), which the developer faithfully implemented and the test-writer faithfully asserted. With `justify-content: flex-end` and `padding: 7px`, the visible's right edge sits at `wrapper.right - padding-right` (inside the wrapper's content box), so the offset between `wrapperLeft` and `visible.right` is `DOT + padding`, not `DOT + 2 * padding`. The implementation as committed at `13644da`/`2a53b42` placed the visible's right edge 5px INSIDE the table instead of 2px past it.

### Attempt 2 — direct fix by orchestrator
- **Fix (Opus, in place):** Replaced `(TOGGLE_DOT_PX + 2 * TOGGLE_HIT_PAD_PX)` with `(TOGGLE_DOT_PX + TOGGLE_HIT_PAD_PX)` in `positionToggle`'s `wrapperLeft` so both axes use the same symmetric form. Updated the two position tests to assert the corrected formula. Added two **cross-check assertions** that the resulting `visible.right` and `visible.bottom` land at the spec-defined edges (`rect.right + TOGGLE_DOT_OVERHANG_PX` and `rect.top + TOGGLE_DOT_OVERLAP_PX`, respectively); these would have caught the math error independent of any formula assertion. The deviation from plan §3.6 is intentional — the plan has a math error and is immutable post-merge.
- **Test run:** `node chrome-extension/tests.js` → **Passed: 526 / Failed: 0** (524 from Attempt 1's test-writer step plus the 2 new cross-check assertions).
- **Reviewer verdict:** APPROVE.

The geometry, DOM shape, accessible name, color, mouse/touch/pen/keyboard interaction, auto-collapse timer, tap-outside collapse, constant interpolation, and `aria-pressed` state-reflection are all covered by tests derived from the spec rather than the implementation. Scope respected — only `chrome-extension/content.js` and `chrome-extension/tests.js` modified; no version files touched; the round/restore pipeline (`runToggleAction`, `roundTable`, `toggleOriginalValues`, `isTableRounded`), the `MutationObserver`/`ResizeObserver` plumbing, and propagation-stoppers are unchanged.
