Linked slider choices

Each card shows a live, working sidebar simulation for the new largest numbers / all other numbers sliders. Discrete stops are 1, 0.75, 0.5, 0.25, 0, -0.25, -0.5, -0.75, -1 — no labels. Bottom slider starts greyed and linked; the moment you touch it, it decouples and stays independent. The results panel under each card shows what two top-band values (27,136, 18,750) and smaller-magnitude values (4,080, 312, 56) would round to with the current slider positions.

Bottom thumb renders as a small tab below the track (rounded only on the bottom corners) so it's always grabbable. While linked it is grey; once you grab it, it turns rust and stays independent. If you later release either thumb on the same stop as the other, they re-link automatically (tab turns grey again). No chip, no hidden affordance — the colour and position carry the state.

F. Step lives with each example, not with the slider

Same mechanics and layout as E, but the bracketed step moves from the slider readout to each example. With largest numbers = 0 the top band reads 30,000 (10k) and 20,000 (10k); with all other numbers = −0.25 the bottom reads 4,000 (250), 300 (25), 55 (2.5) — each value annotated with the step that produced it. Arrows and brackets are aligned via a shared grid so the eye reads down each column.

Math: same engine for all three. current_mag = floor(log10(|n|)); for fractional offset x, step = |x − trunc(x)| · 10^(current_mag + ceil(x)), then Feature-2 floors at 10^current_mag so a value never collapses to zero. Result rows update live as you drag.
